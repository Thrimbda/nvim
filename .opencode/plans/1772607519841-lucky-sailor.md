# Goal
Explain why Org timestamps show Chinese weekdays (e.g. `[2026-03-03 二 22:11]`) and define where to change settings so they render English weekdays.

# Findings
- `nvim-orgmode` timestamps come from `%U` / `%t` templates in `lua/plugins/orgmode.lua`.
- Weekday text is generated in plugin code via `os.date('%a')` (`~/.local/share/nvim/lazy/orgmode/lua/orgmode/objects/date.lua`).
- `%a` is locale-dependent (`LC_TIME`/`LANG`), and this repo currently does not override locale in Neovim startup.

# Recommended Approach
1. Prefer locale fix at process/shell level (lowest risk): set `LC_TIME=en_US.UTF-8` before launching Neovim.
2. If user wants Neovim-only behavior, set locale early in repo config:
   - Modify `init.lua` before `require("config.lazy")`.
   - Add: `vim.env.LC_TIME = "en_US.UTF-8"` (optional fallback `vim.env.LANG = "en_US.UTF-8"`).
3. Do not modify vendor plugin files under `~/.local/share/nvim/lazy/orgmode/...`.

# Files In Scope
- `init.lua` (only if implementing Neovim-local fix)
- `lua/plugins/orgmode.lua` (reference only; no change expected)
- `~/.local/share/nvim/lazy/orgmode/lua/orgmode/objects/date.lua` (reference only; no change)

# Verification
1. In Neovim run: `:lua print(os.date("%a %Y-%m-%d"))` and confirm weekday is `Mon/Tue/...`.
2. Insert an Org timestamp via existing capture templates (`%U`/`%t`) and confirm output style like `[2026-01-23 Fri 13:45]`.
3. Restart Neovim and repeat checks to ensure it persists across sessions.
